## Обзор

FOM Analytics — это полноценное веб-приложение, построенное на современном, продуманном стеке. Цель — минимальный набор зависимостей: только то, что необходимо. Это упрощает поддержку системы и сокращает поверхность атаки.

## Бэкенд

| Слой | Технология |
|---|---|
| Язык | PHP 8.2+ |
| Фреймворк | Laravel 12 |
| Роутинг (фронтенд) | Inertia.js 2 |
| Обработка таблиц | PhpSpreadsheet |
| Типобезопасная привязка маршрутов | Laravel Wayfinder |

Сервер не рендерит HTML напрямую. Весь интерфейс управляется через **Inertia.js**, который связывает контроллеры Laravel с React-компонентами без отдельного API-слоя.

**PhpSpreadsheet** (`phpoffice/phpspreadsheet`) используется на сервере для обработки загружаемых Excel-файлов перед попаданием данных в систему.

## Фронтенд

| Слой | Технология |
|---|---|
| Язык | TypeScript |
| UI-фреймворк | React 19 |
| Библиотека компонентов | shadcn/ui (примитивы Radix UI) |
| Стилизация | Tailwind CSS v4 |
| Таблицы | TanStack Table |
| Графики | Recharts |
| Drag-and-drop | dnd-kit |
| Работа с датами | date-fns |
| Уведомления | Novu |
| Сборщик | Vite 7 |

Компоненты shadcn/ui **копируются непосредственно в кодовую базу**, а не устанавливаются как зависимость. Это означает, что обновления нужно применять вручную — см. [примечание о безопасности компонентов](#component-security) ниже.

## Безопасность

### Слои аутентификации

FOM Analytics использует **Laravel Fortify** для управления аутентификацией. Доступны два уровня защиты:

<Columns cols={2}>
  <Card title="Сессионная аутентификация" icon="lock">
    Стандартные защищённые маршруты. Пользователи входят по email и паролю и получают безопасную серверную сессию.
  </Card>
  <Card title="Двухфакторная аутентификация (2FA)" icon="shield-halved">
    Более строгие маршруты дополнительно требуют TOTP-код. Fortify управляет процессами подключения, проверки и восстановления.
  </Card>
</Columns>

### В будущем: WorkOS AuthKit

Текущая реализация на базе Fortify будет перенесена на **WorkOS AuthKit**, что даст:

- Корпоративное управление идентификацией
- Детальный контроль доступа пользователей и ролей
- Полные журналы аудита всех действий пользователей
- Улучшенные возможности SSO и MFA

Эта миграция запланирована как отдельный этап и не изменит интерфейс приложения.

### Безопасность компонентов

<Warning>
  Компоненты shadcn/ui не обновляются автоматически менеджером пакетов — они хранятся непосредственно в исходном коде. Патчи безопасности для лежащих в основе примитивов Radix UI необходимо применять вручную. Периодически проверяйте релизы [Radix UI](https://github.com/radix-ui/primitives/releases) и [журнал изменений shadcn/ui](https://ui.shadcn.com/docs/changelog) и обновляйте затронутые файлы компонентов.
</Warning>

## Python-скрипты

Небольшая утилита на Python запускается внутри **виртуального окружения** и конвертирует Excel-файлы в формат CSV перед передачей данных в основной конвейер обработки. Она не является частью веб-приложения и запускается как отдельный скрипт.

```
scripts/
└── venv/               # Виртуальное окружение Python (не коммитится)
└── excel_to_csv.py     # Конвертирует .xlsx / .xls в .csv
```

Держите виртуальное окружение изолированным и никогда не устанавливайте пакеты в системный интерпретатор Python.

## Инструменты разработки

| Инструмент | Назначение |
|---|---|
| Laravel Pint | Стиль PHP-кода (PSR-12) |
| ESLint + Prettier | Линтинг и форматирование JS/TS |
| PHPUnit | Тестирование PHP |
| Laravel Sail | Локальная среда на Docker (опционально) |
| Laravel Pail | Просмотр логов в реальном времени |
| Concurrently | Запуск сервера, очереди, логов и Vite в одном терминале |

## Развёртывание

<Note>
  Поскольку стек включает оба рантайма — PHP и Python, можно собрать Docker-образ, который объединяет всё необходимое. Это позволит разработчикам стабильно развёртывать FOM Analytics в любой среде без дополнительных настроек.
</Note>
